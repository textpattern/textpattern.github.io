Common global attributes

These attributes are shared by all tags - and plugins, provided they do not override the functionality.

{% unless include.break == "" %}
break="value"
Where value is an HTML element, specified without angle brackets (e.g. break="li") or some string to separate list items.
Default: {% if include.break == null %}br{% else %}{{ include.break }}{% endif %} (see break cross-reference).
{% endunless %} {% unless include.breakby == "" %}
breakby="integer(s)" v4.7.0+
Used to group list items when separating by break. Possible values are lists of integers, like 2 (groups of 2 items) or 1,2 (alternate groups of 1 and 2 items). Otherwise, the value is evaluated on each loop and break happens as soon as it changes. Note that string must be enclosed in *double* quotes.
Default: {% if include.breakby == null %}1 (unset){% else %}{{ include.breakby }}{% endif %} (see breakby cross-reference).
{% endunless %} {% unless include.breakform == "" %}
breakform="form name"
A form to be used as break, generally jointly with breakby attribute. The special <+> pattern in this form will be replaced with the list "chunk" accumulated when break happens.
Default: {% if include.breakform == null %}unset{% else %}{{ include.breakform }}{% endif %} (see breakform cross-reference).
{% endunless %}
class="class name"
HTML class to apply to the wrapper.
Values: Any valid string.
Default: {% if include.class == null %}unset{% else %}{{ include.class }}{% endif %}.
default="value"
The value to display if the tag's output is blank.
Values: Any.
Default: {% if include.default == null %}unset{% else %}{{ include.default }}{% endif %}.
{% unless include.escape == "" %}
escape="list, of, transforms"
The transforms to apply to the output. See tag escaping for details.
Values: db, float, html, integer, js, json, lower, number, ordinal, quote, [r|l]trim, spell, tags, textile, tidy, title, upper, url, _any tag_
Default: {% if include.escape == null %}html{% else %}{{ include.escape }}{% endif %}.
{% endunless %}
evaluate="number list"
Alter the processing order of top-level tags inside a form/container. Tags are normally executed from top to bottom. If you wish to change this order, imagine Textpattern tags are numbered incrementally from 1 inside a container. To process tags 3 and 5 first, then 1, 2, and 4, specify evaluate="3, 5" in your container tag.
Values: Any comma-separated numbers corresponding with the numerical position of primary (top-level) tags inside the container.
Default: {% if include.evaluate == null %}unset{% else %}{{ include.evaluate }}{% endif %}.
html_id="value"
HTML id to apply to the wrapper.
Values: Any valid string.
Default: {% if include.html_id == null %}unset{% else %}{{ include.html_id }}{% endif %}.
label="value"
The label to display before the output.
Values: Any valid string.
Default: {% if include.label == null %}unset{% else %}{{ include.label }}{% endif %}.
labeltag="tag"
The tag (without angle brackets) to wrap the label.
Values: Any valid HTML tag, e.g. h3 or div.
Default: {% if include.labeltag == null %}unset{% else %}{{ include.labeltag }}{% endif %}.

limit="number" v4.8.8+
The maximum number of items to display per page in lists.
Values: Any integer.
Default: {% if include.limit == null %}unset{% else %}{{ include.limit }}{% endif %} (see limit cross-reference).

not="boolean"

{% endunless %} {% unless include.offset == "" %}
offset="number" v4.8.8+
Apply the given pagination offset to the (list of) things being iterated. May be set by URL parameters.
Values: Any integer.
Default: {% if include.offset == null %}0{% else %}{{ include.offset }}{% endif %} (see offset cross-reference).
{% endunless %} {% unless include.sort == "" %}
sort="column direction" v4.8.8+
Sort the output by the given column(s) and order direction(s).
Values: Specify any valid column name{% if include.sortvals != null %} from the ones given below{% endif %}, followed by a space and then either asc or desc to sort in ascending or descending order, respectively. Use rand() for random sorting order. {% if include.sortvals != null %}{{ include.sortvals }}{% endif %}
Default: {% if include.sort == null %}0{% else %}{{ include.sort }}{% endif %} (see sort cross-reference).
{% endunless %}
replace="string"
Replace the content stripped via the trim attribute.
Values: Any valid string.
Default: {% if include.replace == null %}unset{% else %}{{ include.replace }}{% endif %}.
trim="string or regex"
Remove the matching patterns from the output.
Values: Any. Use /value/ to trigger regular expression matching, otherwise the value will be treated verbatim as a set of characters to trim.
Default: {% if include.trim == null %}unset{% else %}{{ include.trim }}{% endif %}.
{% unless include.variable == "" %}
variable="value" v4.9.0+
Assign the output of the tag to the named variable, instead of displaying it.
Values: Any valid string to be used as a variable name.
Default: unset.
{% endunless %}
wrapform="form name"
The form to be used as wraptag. Handy if the wraptag pattern is too long, or is reused.
Values: Any defined form name.
Default: {% if include.wrapform == null %}unset{% else %}{{ include.wrapform }}{% endif %}.
wraptag="tag or pattern"
The tag (without angle brackets) to wrap the output.
Values: HTML tag or a string containing <+> pattern that will be replaced by the output.
Default: {% if include.wraptag == null %}unset{% else %}{{ include.wraptag }}{% endif %}.
